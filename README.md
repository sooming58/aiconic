Ⅴ. 서비스 소개 

 

ⅰ. 서비스 개요 

 

 이 서비스는 시각장애인을 포함해 행정이나 금융 서류를 혼자 작성하기 어려운 사람들을 위해 개발된 웹 기반 문서 작성 플랫폼이다. 사용자는 원하는 문서 템플릿을 선택한 뒤 음성으로 필요한 정보를 입력한다. AI가 자동으로 필요한 정보를 감지하고 추출하여 암호화된 형태로 안전하게 처리한다. 이렇게 수집된 정보를 토대로 자동으로 문서를 작성하고 PDF 파일로 변환해 제공한다. 또한 플랫폼은 음성 안내 기능을 지원하여 시각장애인도 쉽게 사용할 수 있도록 설계되었으며, 이를 통해 사용자는 타인에게 개인정보를 노출하지 않고도 독립적으로 행정, 금융 서류를 작성할 수 있다. 결과적으로 이 서비스는 접근성을 강화하고 개인정보 보호를 실현하며, 문서 작성 과정의 효율성을 높여 사용자에게 자율성과 편의성을 동시에 제공한다.  

 

ⅱ. 타 서비스와의 차별점 

1 

  

현재 공공기관에 문서를 출력할 수 있는 키오스크가 있다. 하지만 이조차도 시각장애인과 노인 등 행정이나 금융 서류를 혼자 작성하기 어려운 사람들에겐 큰 부담이 된다. 직접 공공기관을 방문해야 한다는 점부터 큰 과제일 수 있다. 키오스크는 음성 기반 서비스는 가지고 있더라도 사용자가 직접 모든 내용을 입력해야 하기에 혼자서 서류를 출력하기엔 쉽지 않다. 

 하지만 우리 서비스는 사용자가 음성으로 정보를 입력하면, AI가 필요한 데이터를 자동으로 추출하고 이를 암호화하여 외부에 노출되지 않도록 안전하게 처리한다. 또한, 행정, 금융 서류 템플릿을 제공해 복잡한 형식을 사용자가 직접 맞출 필요 없이 손쉽게 문서를 완성할 수 있다. 그렇기에 번거롭게 공공기관을 찾아갈 필요도 없어진다. 따라서 개인정보 보호와 접근성, 자동화된 문서 생성이라는 세 가지 핵심 가치를 결합하여 사용자가 타인의 도움 없이 독립적으로 행정 및 금융 업무를 처리할 수 있도록 지원한다. 

 

ⅲ. 구현 내용 및 결과물 

 

 우리 서비스는 사용자가 원하는 서류 템플릿을 선택한 뒤 필요한 정보를 음성으로 안내받을 수 있도록 설계되어 있다. 사용자가 개인정보를 음성으로 입력하면 시스템이 이를 자동으로 텍스트로 변환하고, AI가 분석하여 선택된 문서에 필요한 내용을 추출한다. 변환된 개인정보는 즉시 암호화되어 저장되며, 문서 생성 과정에서 필요한 순간에만 복호화 되어 활용되므로 외부 노출 위험을 최소화할 수 있다. 이후 사용자가 선택한 템플릿에 입력값이 자동으로 삽입되어 문서가 완성되며, 작성된 서류는 즉시 PDF 파일로 변환되어 다운로드하거나 제출용으로 활용할 수 있다. 또한 시각장애인을 위해 대형 버튼, 간단한 단계 안내, 음성 중심 UI를 제공하여 시각적 인터페이스에 의존하지 않고도 독립적으로 서비스를 이용할 수 있는 접근성 최적화 환경을 마련하였다. 

 

ⅳ. 구현 방식 

 

음성 입력 및 텍스트 변환 

 

 streamlit-webrtc를 활용하여 브라우저에서 오디오를 스트리밍하고, 노이즈 억제와 자동 게인 제어 같은 옵션을 적용해 음질을 최적화하였다. 사용자가 음성 녹음을 하면 오디오 프레임이 pydub.AudioSegment 객체로 변환되어 세션 상태에 버퍼로 저장되며, 녹음이 종료되면 지정된 폴더에 .wav 파일로 저장된다. 이렇게 생성된 파일은 시각적 인터페이스에서 바로 확인할 수 있도록 관리된다. 세션 상태에 녹음 파일 경로를 저장하여 일관성을 유지하고, 저장된 파일을 불러와 웹 환경에서 바로 재생할 수 있도록 하였다. 

 녹음된 음성은 OpenAI의 Whisper 모델을 통해 텍스트로 변환된다. Whisper.load_model(“small”)을 사용하여 소형 모델을 불러와 변환 속도와 정확성을 균형 있게 유지하였으며, 변환 결과는 웹 인터페이스에 출력되어 사용자가 즉시 확인할 수 있다. 마지막으로 오류 처리 단계에서는 녹음 파일이 존재하지 않을 경우 변환 버튼 클릭 시 오류 메시지를 출력하여 사용자가 먼저 녹음을 진행하도록 안내하고, 변환 성공 여부에 따라 직관적인 피드백을 제공하였다. 

 

GPT(OpenAI) 기반 텍스트 분석 및 문서 자동 생성 기능 

 

 음성 입력을 통해 받은 텍스트에서 GPT(gpt-4o)가 개인정보를 자동 추출하고, 이 정보를 JSON 구조로 정리해 문서 생성에 활용된다. 문서 유형에 따라 전문적 내용을 생성하는 프롬프트를 사용하며, 이를 통해 행정, 금융 서류와 같은 복잡한 문서도 손쉽게 작성할 수 있다. 시각장애인을 위한 음성 기반 서비스가 제공되어 문서 전체 내용을 시스템이 음성으로 안내해준다. 또, 사용자가 “본인은 상기 내용을 확인하고 이에 동의합니다.”라고 말하면 서명으로 처리한다. 이때 서명은 서명 시각(Timestamp), 문서 내용 기반 SHA-256 해시값, 녹음된 음성 파일의 URL, 동의 음성 파일을 포함함 QR 코드이며, 이를 문서에 삽입해 전자 문서의 무결성, 서명자 확인, 서명 시점 검증이 가능해진다. 이렇게 자동으로 생성된 문서 내용을 PDF로 변환하여 출력이 가능하다. 결과적으로 시각적 인터페이스 없이도 문서 생성, 검토, 서명, PDF 출력까지 전 과정을 수행할 수 있어 정보 취약계층이 독립적으로 안전하게 문서를 작성할 수 있도록 지원한다. 

 시각장애인을 위한 음성 안내 및 전자 서명 기능을 지원하기 위해 생성된 음성 파일을 안정적으로 저장하고, 모바일 기기에서 QR 코드 스캔을 통해 해당 파일에 접근할 수 있도록 Amazon Web Services S3(Simple Storage Service)를 활용하여 백엔드 인프라를 구축하였다. Python의 boto3 라이브러리를 사용하여 AWS S3 클라이언트를 초기화하고, 보안을 위해 Access Key ID와 secret Access Key는 환경 변수를 통해 관리하였다. 이후 s3.client.upload_file() 메서드를 활용하여 로컬에서 생성된 .wav 파일을 지정된 S3 버킷에 업로드하였다. 버킷 정책(Bucket Policy)을 설정하여 모든 사용자에게 s3:GetOObject 권한을 부여함으로써 파일이 공개적으로 액세스 가능하도록 하였으며, 업로드가 완료된 후에는 파일의 고유 퍼블릭 URL을 생성하였다. 이 URL은 최종적으로 QR 코드에 인코딩되어 문서와 연결되며, 사용자가 QR 코드를 통해 음성 파일을 직접 확인할 수 있도록 한다. 

 

개인정보 암호화 

 

본 서비스는 음성으로 입력 받은 개인정보(이름, 주소, 주민등록번호 등)을 다루므로, 데이터의 안전한 처리와 보관이 중요한 과제입니다. 따라서 사용자는 별도의 절차 없이 편리하게 서류를 작성하면서도, 입력된 민감 정보가 시스템 내애서 강력하게 보호되도록 “투명한 백엔드 자동 암호화”를 목표로 설계하였습니다. 

파이썬의 대표적인 암호 라이브러리인 cryptography를 사용하였습니다. Cryptography 라이브러리 중에서도 Fernet 프로토콜을 사용하여 대칭키 방식의 암호화를 진행하였습니다. Fernet은 AES-128-CBC(알고리즘)와 HMAC-SHA256(데이터 무결성 검증 기능) 등의 기술들을 결합한 표준화된 프로토콜입니다. 

암호화 과정은 사용자가 신경쓰지 않더라도, Streamlit의 세션 상태 내에서 이루어집니다. Fernet Key를 자동 생성하고, 음성 입력 텍스트를 가져와 Key로 자동 암호화를 하여 암호문 상태로 시스템 내부에서 저장합니다. 그리고 PDF문서 작성 미리보기 시에 개인정보 유출을 방지하기 위해 추가적으로 암호화된 상태의 자신의 개인정보를 제공합니다. 이는 사용자가 시각장애인 등을 대상으로 한다는 점에서 추가한 보안 화면 표시 정책입니다. 만약 사용자가 체크박스를 누를 경우에만, 시스템이 내부 키를 사용하여 일시적으로 복호화하여 보여줍니다. PDF 생성 시에는 세션 메모리에 별도 저장된 복호화된 원본 데이터를 사용하여 서류의 내용이 원하는 한글로 정상 출력되도록 보장합니다. 

즉, Fernet 프로토콜을 통해 개인정보는 입력과 동시에 안전하게 보호되며, 사용자에게는 불필요한 키 관리 부담을 주지 않으면서도 최종 서류는 정상적으로 작성되는 개인정보 보안성 + 사용 편의성 둘 다 모두 확보한 아키텍처를 구현하려고 목표하였습니다. 

 

UI 설계 

 

UI는 접근성 중심을 기반으로 설계되었다. Streamlit 라이브러리을 사용해 전체 인터페이스는 복잡한 시각 요소를 최소화하고, 단계별로 AI가 음성으로 안내해줘 보다 더 직관적으로 이해할 수 있도록 구성하였다. gTTS 라이브러리를 사용하여 텍스트를 음성(mp3) 파일로 변환한다. Streamlit에서 버튼 클릭 시 즉시 실행되도록 HTML <audio> 태그를 활용하였고, 파일을 base64로 인코딩하여 <audio autoplay> 태그에 삽입하였다. 

 

Ⅵ. 향후 개선 혹은 발전 방안 

 

 현재 서비스는 시각장애인과 고령층 등 스스로 행정·금융 서류를 작성하기 어려운 이용자들을 실질적으로 지원하는 데 초점을 두고 있다. 그러나 본 프로젝트는 단기적 기능 제공에 그치지 않고, 장기적으로는 다양한 환경과 사용자 요구에 대응할 수 있는 확장 가능한 구조를 지향한다. 우선 모바일 앱 형태로의 확장을 통해 접근성을 강화하고, 실시간 음성·텍스트 처리 속도를 최적화하여 사용자 경험을 향상시키고자 한다. 또한 공공기관의 전자문서·민원 API와의 연동을 추진함으로써 서류 발급, 제출, 확인까지 하나의 플랫폼에서 가능한 통합 행정지원 시스템으로 발전시킬 수 있다. 이와 동시에 보안성을 높이기 위해 문서 암호화, 개인정보 비식별화, 접근 제어 등 고도화된 보안 기능을 단계적으로 도입하여 안전한 서비스 환경을 구축할 예정이다. 향후 개발 과정에서는 대학 SW 중심대학 사업단, 학과 교수진의 기술 멘토링, 개발 동아리와의 협업 등 다양한 내부 자원을 적극 활용하여 기술적 완성도를 높이고, 실제 시각장애인 단체와 복지관을 대상으로 한 테스트와 피드백 수렴을 통해 서비스의 실효성을 검증할 계획이다. 이러한 개선 과정을 통해 단순 보조 도구 수준을 넘어 사용자 개개인의 상황과 필요에 맞춘 맞춤형 지능형 문서 작성·관리 플랫폼으로 단계적으로 발전시키고자 한다.
